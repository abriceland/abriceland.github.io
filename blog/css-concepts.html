CSS Awakening - Display Inline vs Inline Block?

This week I was initiated into the world of CSS - Cascading Style Sheets. At first, I was delighted. Break out formatting from the outline and content in an HTML file - what a brilliant concept! The process of coding in CSS is simple, so I was unprepared for what was right around the corner.

CSS is hard, very hard. There are hundreds of possibilities for sizing and positioning things, changing how text looks, adding images, creating extra features, etc. It isn't something for the faint of heart. The first big issue I had was trying to place to sections right next to each other.

I tried and tried to get the second section to move up. I researched, but to no avail. Finally, I ran across the display: inline-block value and I was saved.

As an inline-block, it moved up, not into the right place, but there was movement. It was beside the other section at last! What had created this magic moment?

Blocks, which are another alternative in display, take up the whole width of a site. Even if they are off to one side, nothing is allowed to sit next to them.

In the example above from my website, the navigation bar (which is what we use to move to different pages on a website) on the left is set as inline and not allowing the element below to be on the same line.

In this second example, the sections are on the same line, still separate blocks, but sharing the website sandbox. A command called float used to be used to place elements on website, but inline-block is an upgrade - "the new black".

The actual coding is simple compared to the mess your website can become without these commands.

p {display: inline;} compared to p {display: inline-block}

Because of the rules governing CSS, most of which are still a mystery to me, understanding inline vs. inline block won't solve all of your problems. But it's a start.
