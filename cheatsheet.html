Ruby Cheatsheet

As a notetaker, I've been trying my best to document important points for myself along my Dev Bootcamp journey. Below are a few reminders for the Ruby programming language.

  • Method

    Method is a procedure that allows you to extract the common code to one place

    Argument is a piece or pieces of information that are sent to a method to be modified or used to return a specific result. We "pass" arguments to a method when we call it.

    def method(argument)

  • Numbers

    Integers are whole numbers – 1 99, 134, -6

    Floats are decimals – 1.1, -55.9, 2,222.2

  • Strings

    Strings are words or text, including punctuation, digits, symbols and spaces. They can also be empty. They are written in quotes like below:

    "Hi, I'm in quotes, so I'm a string!"

    You can’t add a number to a string. The number has to be changed to a string first.

    Use \ in place of an apostrophe - it\s the escape character! \\ is same as \ - it escapes itself.

  • Converting String, Integers & Floats

    .to_i - converts to integer

    .to_f - converts to float

    .to_s - converts to string

  • Some Operators

    + adds values on either side of the operator

    - subtracts right hand operand from left hand operand

    * multiplies values on either side of the operator

    / divides left hand operand by right hand operand

    % divides left hand operand by right hand operand and returns remainder. Called modulus

    ** performs exponential (power) calculation on operators

    == checks if the value of two operands are equal or not. If yes, then condition becomes true

    != checks if the value of two operands are equal or not. If values are not equal, then condition becomes true

    > checks if the value of left operand is greater than the value of right operand. If yes then condition becomes true

    < checks if the value of left operand is less than the value of right operand. If yes, then condition becomes true

    >= checks if the value of left operand is greater than or equal to the value of right operand. If yes then condition becomes true

    <= checks if the value of left operand is less than or equal to the value of right operand. If yes then condition becomes true

    <=> returns 0 if first operand equals second, 1 if first operand is greater than the second and -1 if first operand is less than the second.

    === used to test equality within a when clause of a case statement.

    .eql? true if the receiver and argument have both the same type and equal values.

    equal? true if the receiver and argument have the same object id.

  • Returning and Outputting

    p returns the evaluated code AND print to the console

    puts returns nil and prints the output to the console with a new line

    print returns nil and prints the output to the console

  • Interpolation

    #{ } - interpolates or embeds in string form the results of a variable or number

  • Code Blocks

    Code blocks can be written two ways. For short blocks of one line, use:

    {|number1, number2| number1 + number2}

    For multiple lines within a block, use: 

    
method.each do |argument|
	do something fancy here
end

  • Looping

    Looping is an iterator. It just needs a block providing syntax

    While, until and for are all loops in Ruby, though for is rarely used

    Another way to loop is with times. Times is an instance method of the integer class. See the example below:

    5.times

  • Arrays

    Arrays are unordered lists of objects or empty placeholders for lists. They are contained within square brackets, as in the example below:

    array = [4, 8, 12, 16, 20]

    Arrays are interated over to pick up each item and do something to it, as in the block below:

    array.each {|e| puts “The block just got handed #{e}.”}

  • Hashes

    Hashes are lists of objects with relationships - keys and values. They are contained within curly brackets with the key always listed first and value second.

    hash = {"cat1" => "Sylvia", "cat2" => "Paro", "cat3" => "Audrey"}
